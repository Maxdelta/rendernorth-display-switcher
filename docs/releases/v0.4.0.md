# RenderNorth Environments v0.4.0 — Private Beta

RenderNorth Environments lets you save and activate named Windows display workspaces for gaming, streaming, development, presentations, and more.

## Private Beta Release

This Windows 11 x64 build was released as a GitHub prerelease on 2026-07-23. It is an Early Access private beta, not a stable public v1 release.

- Release: https://github.com/Maxdelta/rendernorth-display-switcher/releases/tag/v0.4.0-private-beta

## Included

- Capture, save, manage, and activate named Windows display environments.
- Detect the active environment from display topology, positions, dimensions, rotation, and scaling.
- Launch saved environments silently from stable Stream Deck shortcuts.
- Retain legacy Game Mode and Script Mode launcher compatibility.
- Create additional environments without losing the persistent creation card.
- Preserve saved environments and managed shortcut registrations in local user data.
- Keep installed user data outside the installer-managed application directory so repair, upgrade, and reinstall preserve it.
- Check for updates from the existing configured release source.
- Identify connected display targets and open Windows Display Settings.

## Private Beta Fixes

- Stabilized responsive header, hero, Welcome, Quick Actions, and status layouts.
- Removed non-interactive empty-space hover reactions.
- Fixed saved-environment cards disappearing during refresh.
- Fixed environment icon ellipsis artifacts.
- Corrected active-environment detection when profiles share the same monitor IDs.
- Clarified the hero card with an explicit Active state.

## Notes

- Windows 11 x64 only.
- Displays are the first supported environment capability.
- Display profiles may need to be recaptured after GPU-driver, docking, or monitor-enumeration changes.
- This private installer is not code-signed and may show a Windows security prompt.
- Windows may identify the installer as Unknown Publisher or display a Microsoft Defender SmartScreen warning.
- Minor visual-polish opportunities remain.
- The repository, executable, and package retain the legacy Display Switcher identity for v0.4.x compatibility.
- Broader public launch and website work are separate tasks.
