## What is an Environment?

An Environment is a saved workspace for what you are about to do. Gaming can use one display arrangement, Streaming can route a different screen to your capture card, Development can use an extended multi-monitor layout, and Presentation can simplify the setup for recording or sharing.

## Feature sections

### Named workspaces

Create, rename, duplicate, reorder, and delete environments that match how you work.

### Capture and activate

Capture the current Windows display setup, then restore it later with one click.

### Stream Deck workflow

Create stable environment shortcuts and trigger them silently while the application window stays closed.

### Local-first privacy

Profiles, logs, and settings remain on the local Windows account. No cloud account is required.

## Migration

Existing RenderNorth Display Switcher users can update without losing saved Game Mode and Script Mode configurations. Those configurations are migrated into named environments and remain compatible with existing launchers during the transition.

## FAQ

**Does it require Stream Deck or Elgato?** No. They are optional integrations and use cases.

**Can I create more than two environments?** Yes. Environments are named and independently managed.

**Does it change audio or launch apps?** Not in v0.4.0. The early-access release manages display configurations.

**Where is data stored?** In the local RenderNorth application data folder for the current Windows account.

**Does it work offline?** Yes. Activation is local. Internet access is only needed to check for updates.

**What happens after a GPU driver or docking change?** Recapture the affected environment if Windows re-enumerates the displays.

**How do I report a bug?** Use the GitHub issue tracker and include the application version and relevant local log excerpt without personal information.
